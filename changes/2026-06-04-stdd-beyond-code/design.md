### 1.3 关键论证链

```
AI编程方法论演进(SDD共识)
  → STDD在共识之上的独特贡献(强制TDD+经验自学习)
  → 这两个贡献的本质不依赖"代码"介质
  → Agent验证管线提供了非代码验证的技术基础
  → 金融/日常/运维 12个场景的实证分析
  → 定量效果评估 + 定性价值分析
  → 适用边界与未来方向
  → 结论：STDD是通用的AI执行质量保障框架
```

## 二、PPT 结构设计

### 2.1 规模

30 页，分为 6 个章节组。

| 组 | 页数 | 内容 |
|----|:---:|------|
| 封面与概述 | 4 | 封面/核心论点/关键数字/目录 |
| 方法论 | 5 | 三要素/六阶段/经验复利/双轨制 |
| 技术 | 4 | Agent管线/assertion类型/对比 |
| 场景·金融 | 6 | 4个金融场景深度展示 |
| 场景·日常运维 | 5 | 官网/文档/部署/新人/客户 |
| 评估与结论 | 6 | 效果数据/对比/边界/结论/致谢 |

### 2.2 设计规范

- 主题：延续 ppt/index.html 的靛蓝瓷 (Indigo Porcelain) 配色
- 字体：Playfair Display + Noto Serif SC + IBM Plex Mono
- 动画：motion.js 渐入效果
- 响应式：支持 1920×1080 到 768×1024

## 三、质量验证方案

### 3.1 论文质量 CP（agent_spec.yaml）

| CP | 检查项 | 验证方式 |
|----|--------|---------|
| CP-1 | 字数达标（≥30,000） | `wc -m` 统计 |
| CP-2 | 章节完整（10章+摘要+参考文献） | grep 标题 |
| CP-3 | 引用数量（≥30） | grep `\[` 统计 |
| CP-4 | 内部一致性（章节互引用） | 人工审校 |
| CP-5 | 拼写与语法 | AI 辅助检查 |
| CP-6 | 数据真实性（案例数据可查证） | 对照源码 |

### 3.2 PPT 质量 CP

| CP | 检查项 | 验证方式 |
|----|--------|---------|
| CP-7 | 页数达标（≥25） | 统计 `<section>` 数量 |
| CP-8 | 覆盖论文全部章节 | 逐章对照 |
| CP-9 | 浏览器兼容 | Chrome/Firefox/Edge 截图 |
| CP-10 | 响应式适配 | 375/768/1920 宽度测试 |

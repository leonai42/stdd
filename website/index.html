六阶段流程

从需求到交付，每阶段都有明确产出物与质量门

Phase 1

UNDERSTAND · 需求理解

模糊需求 → proposal.md 可验证提案

Gate 1 · 强制确认
Phase 2

SPEC · 规格设计

design.md + specs + test-plan

Gate 2 · 强制确认
Phase 3

SLICE · 切片规划

tasks.md + slices.md

Gate 2 后可自动
Phase 4

BUILD · TDD 实现

RED→GREEN→REFACTOR

偏离自动记录
Phase 5

VERIFY · 质量验证

全量测试 + 12类检查

Gate 3 · 强制确认
Phase 6

DELIVER · 交付

归档 + 合并 + tag

自动执行

V2.5 核心能力

不只是流程，更是一套会进化的质量保障体系

🧠
自学习经验库5 态 FSM 生命周期。项目用得越久，AI 越了解你的代码模式。
🌐
社区经验共享池零后端设计。导出脱敏，导入去重。一个人的教训，所有人的经验。
🛡️
12 类失败模式检查幻觉/蔓延/级联/丢失/误用/偏差/断链/质量/衰减/真空/断层/锚定。
🤖
多 Agent 并行适配并行切片派发 + Agent 验证管线。同 change 下多切片并行。
📋
跨 Session 状态恢复phase_context + freshness。Agent 跨 session 一键恢复。
🔑
Gate 三通道确认对话/CLI/Token。Gate 2 后可选长程全自动模式。

快速开始

三步在你的项目中启用 STDD

# 1. 克隆并初始化
git clone https://github.com/leonai42/stdd.git
cd stdd && python bin/stdd init
# 2. 安装到 AI 平台
python bin/stdd install claude-code
# 3. 开始第一个需求
/stdd-understand 你的需求描述

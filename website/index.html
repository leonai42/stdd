六阶段流程

从需求到交付，每阶段都有明确产出物与质量门

Phase 1

UNDERSTAND · 需求理解

产出：proposal.md · 模糊需求 → 可验证提案

Gate 1 · 强制确认
Phase 2

SPEC · 规格设计

产出：design.md + specs/*.md + test-plan.md

Gate 2 · 强制确认（最关键）
Phase 3

SLICE · 切片规划

产出：tasks.md + slices.md · 垂直切片 + 拓扑排序

Gate 2 后可长程自动
Phase 4

BUILD · TDD 实现

RED → GREEN → REFACTOR 循环，逐切片推进

设计偏离自动记录
Phase 5

VERIFY · 质量验证

全量测试 + 覆盖率 + E2E + 11 类故障检查 + Lint

Gate 3 · 强制确认
Phase 6

DELIVER · 交付

归档至 archive/ + 合并 specs + git tag

自动执行

快速开始

三步在你的项目中启用 STDD，支持 6 大 AI 平台

# 1. 初始化 STDD 目录结构
python /path/to/stdd/bin/stdd init
# 2. 安装技能到 AI 平台（以 Claude Code 为例）
python /path/to/stdd/bin/stdd install claude-code
# 3. 开始第一个需求
/stdd-understand 你的需求描述

# 也支持 Cursor、Copilot、Aider、WorkBuddy、Trae
python /path/to/stdd/bin/stdd install cursor
python /path/to/stdd/bin/stdd install copilot
python /path/to/stdd/bin/stdd install aider

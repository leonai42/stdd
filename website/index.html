三档执行模式

轻量/标准/彻底 — 智能适配从 typo 修复到架构重构

0-3

🪶 轻量模式

bug修复/小调整 · 简化SPEC→跳过SLICE→RED→GREEN · TDD底线不跳过

4-7

⚖️ 标准模式

功能增强/中等重构 · 完整六阶段流程 · 经验库全量加载

8-17

🔬 彻底模式

新模块/架构变更 · 标准+pass@k+安全/性能子代理 · 最高质量保障

Phase 1 末尾自动计算6维度复杂度评分，智能推荐模式。核心原则：任何模式下 TDD 底线（RED→GREEN）都不可跳过。

六阶段流程

从需求到交付，每阶段都有明确产出物与质量门

Phase 1

UNDERSTAND · 需求理解

模糊需求 → proposal.md 可验证提案

Gate 1 · 强制确认
Phase 2

SPEC · 规格设计

design.md + specs + test-plan

Gate 2 · 强制确认
Phase 3

SLICE · 切片规划

tasks.md + slices.md

Gate 2 后可自动
Phase 4

BUILD · TDD 实现

RED→GREEN→REFACTOR

偏离自动记录
Phase 5

VERIFY · 质量验证

全量测试 + 12类检查

Gate 3 · 强制确认
Phase 6

DELIVER · 交付

归档 + 合并 + tag

自动执行

V2.9 核心新特性

六大升级，从编程方法论到通用质量保障体系

⚡
三档执行模式Phase 1末尾自动计算6维度复杂度评分（0-17分），智能推荐轻量/标准/彻底模式。
🎯
智能门禁系统四级范围分类器（micro/small/medium/large），分析变更范围给出智能建议。
📦
批次目录管理轻量模式共用批次目录，按时间窗口自动归档。stdd batch CLI命令族。
🔄
版本升级管理stdd upgrade 一键全局升级，upgrade --check 差异检查，upgrade --force 强制同步。
🔬
Plankton + pass@kPlankton三级自动修复（格式化/建议/报告）+ pass@k统计验证检测Spec歧义。
🌐
STDD beyond code从编程方法论扩展为通用质量保障方法论。任何可定义预期结果的任务都适用。

快速开始

三步在你的项目中启用 STDD

# 1. 克隆并初始化
git clone https://github.com/leonai42/stdd.git
cd stdd && python bin/stdd init
# 2. 安装到 AI 平台
python bin/stdd install claude-code
# 3. 开始第一个需求
/stdd-understand 你的需求描述

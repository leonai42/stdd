`stdd hooks install` 将 SessionStart/PreCompact/Stop 三个生命周期 hook 写为字符串格式（`"python .stdd/hooks/xxx.py"`），但 Claude Code 要求 `[{hooks: [{type: "command", command: "..."}]}]` 的 matcher 数组格式。同一文件中的 PreToolUse 用了正确格式，说明缺乏"参照已有正确样例"的编码习惯。

测试 `test_hooks.py` 的断言仅验证 `key in dict`，未检查 value 类型和结构。`stdd hooks status` 只检查脚本文件是否存在，不验证配置有效性。Claude Code 的 `/doctor` 能发现此问题，但未被集成到任何检查流程中。

**教训**：任何"写入配置 → 后续依赖该配置生效"的命令，必须在写入后立即做一次结构校验。测试必须覆盖"输出格式正确"而不只是"输出存在"。

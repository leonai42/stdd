# TDD 规范 / TDD Rules

> 始终在线规则 — Phase 4 BUILD 自动加载

## 核心原则

1. **RED → GREEN → REFACTOR 顺序不可变** — 先写测试，再写实现，最后重构
2. **每个公开函数必须有测试** — 测试验证行为而非实现
3. **测试命名**：`test_<被测方法>_<场景>_<预期结果>`
4. **测试独立**：每个测试不依赖其他测试的执行顺序
5. **测试快速**：单元测试应在毫秒级完成，不应有网络/IO 依赖

## AI 编码时务必遵守

- ❌ 绝不先写实现代码再补测试
- ❌ 绝不跳过失败测试直接写实现
- ✅ 每个 TC-ID 必须对应一个测试函数
- ✅ 测试函数注释中标注 TC-ID

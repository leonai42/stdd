05 / 28
📐

软件质量模型

McCall 三要素 · ISO 25010 八特性 · Humphrey 质量门。STDD的12类失败模式覆盖ISO 25010的6个质量特性。

🔄

过程改进理论

Deming PDCA 循环 · CMMI 五级成熟度 · Six Sigma DMAIC。STDD经验系统=自动化DMAIC循环。

🧠

知识管理模型

Nonaka SECI 模型 · Kolb 学习循环。STDD经验FSM精确实现了外化→组合→内化→社会化的知识创造过程。

⚙️

控制论基础

Wiener 反馈控制 · Kolmogorov 复杂性。Spec=目标值，Test=测量值，Learn=纠偏。控制的有效性不依赖被控对象性质。

👤

人因工程

Parasuraman 自动化层级 · Bainbridge 自动化悖论 · Endsley 情境意识。STDD Gate设计=自适应自动化，关键节点保留人工。

📊

信息与决策论

Spec的精确度与AI输出不确定性的反比关系。Spec锚定法L1-L4本质：逐步增加描述精确度以降低实现不确定性。

steps:
  - id: CP-1
    action: curl /health
    assertions:
      - type: http_status

启示1

质量保障应独立于领域。AI能力同质化→经验积累是护城河。

人机协作的界面设计至关重要。自动化梯度>全自动/全手动二元选择。

启示3

隐性→显性知识转化是组织能力建设的核心。STDD经验FSM=可操作的SECI模型。

Key Findings

研究发现

✅ 可行性成立

STDD的Spec/Test/Learn三要素在非编程场景中完全有效。Agent CP断言是关键桥梁。

✅ 有效性显著

12个场景定量数据: 遗漏率-90%+，文档问题-85%，回滚率-100%，新人上手-60%。

✅ 边界清晰

适用于"可定义预期+可客观验证+有重复性"的任何工作。不适用于纯创意/一次性任务。
